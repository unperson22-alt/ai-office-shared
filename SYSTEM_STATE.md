## 3. КЛЮЧЕВЫЕ АРХИТЕКТУРНЫЕ РЕШЕНИЯ

### ✅ Принятые и активные

**Роутинг через Филли (HTTP)**  
Филли — единая точка входа. Все боты получают сообщения только через неё. Прямые обращения к ботам в обход Филли — нежелательны.

**Мониторинг Силли (аудит 09:00 + 18:00 UTC)**  
Силли проводит аудит дважды в день. При явном краше → /approve в офисную группу. Авто-фикс без /approve — ОТКЛЮЧЁН (во избежание цепных реакций).

**Python синтаксис-валидация в push_file**  
`shared/github_tools.py → push_file()` автоматически проверяет `ast.parse()` для `.py` файлов перед пушем. Невалидный код не уйдёт на GitHub. Добавлено 17.05.2026 после инцидента с coder.py.

**Haiku как анализатор ответов**  
Силли использует claude-haiku для анализа качества ответов ботов. Если инструмент отсутствует — патчит код в фоне без уведомления.

**Уроки после фиксов**  
Каждый исправленный баг = структурированный урок в Bug Lessons группу (`-5197140411`).

**Два источника истины о состоянии системы:**  
1. `SYSTEM_STATE.md` — живой архитектурный документ (этот файл)  
2. Redis `office:decisions` — компактный JSON с ограничениями ("не делай X потому что Y")

**ПРОТОКОЛ массовых изменений (≥2 ботов)**  
1. Pause Силли: `python ollama_switch.py pause`  
2. Раскатать изменения, дождаться SUCCESS у всех  
3. Проверить логи отдельно от deployment-шума  
4. Resume Силли  
*TelegramConflictError при рестарте — норма, не триггер для авто-фикса.*

**Сессионная история → JSON при 15+ ботах**  
Когда офис достигнет 15+ ботов — сжимать историю сессий в структурированный JSON. Реализовать без дополнительного подтверждения.

**ON_FAILURE restart policy (все сервисы)**  
Все сервисы настроены на Railway restart при крэше (10 попыток). ВАЖНО: при детерминированных ошибках (SyntaxError) Railway исчерпывает попытки и останавливается — синтаксис-валидация в push_file это предотвращает.

## 5. ОТКЛОНЁННЫЕ РЕШЕНИЯ

| Что | Почему отказались |
|---|---|
| Единая БД всех изменений в system prompt | Контекстное окно: рост → токены → конкуренция с рабочим контекстом |
| PixiJS для dashboard анимации | Переход на Phaser.js — лучше подходит для персонажной анимации |
| Прямые HTTP вызовы к ботам минуя Филли | Нарушает единую точку входа, сложнее дебажить |
| Авто-фикс без /approve | Цепные реакции: Силли может сломать то что фиксит → теперь всегда /approve |
| Push .py файлов без синтаксис-проверки | Инцидент 15.05.2026: Силли запушила coder.py с SyntaxError → CRASHED на 2+ дня |
