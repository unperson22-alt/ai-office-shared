Группа — Redis вместо RAM
conversation_history перенесена в Redis для всех ботов. Рестарт больше не сбрасывает историю.
ГОТОВО
Redis для группового чата
office:group:history в Redis. Ключ общий для офиса.
ГОТОВО
Передавать контекст группы в process()

Сжатие истории при 15+ ботах
office:group:ctx (TTL 24ч). При 15+ записях Haiku сжимает в сводку fire-and-forget. group_history_get() → [Профиль команды] + [Сводка беседы] + последние 5 сырых.

Профиль группы в целом
office:members (TTL 30д). Haiku строит профиль команды по групповой истории. Инжектится в group_history_get() как [Профиль команды]. Обновление раз в неделю fire-and-forget.

Автоматический бэклог по каждому участнику

Quick Reply кнопки у Крисс

Scheduled сообщения у Крисс
POST /send_scheduled: {chat_id, message, user_id, generate?}. Если generate=true — прогоняет через Claude. Railway Cron дёргает endpoint по расписанию → Крисс шлёт сообщение. Настройка Cron — в Railway Dashboard.

ЗАДАЧИ — УМНЫЙ РОУТИНГ
HTTP роутинг с fallback
Базовый роутинг через internal URLs работает для большинства ботов.
ГОТОВО
Prompt Enhancer перед роутингом
Haiku улучшает запрос до маршрутизации — повышает точность роутинга.
ГОТОВО
Name-map для прямых обращений
"Гослинг, чуешь?" → ГОСЛИНГ минуя LLM-классификатор.
ГОТОВО
Нет проверки health ботов перед роутингом
Закрыто: health-cache в Redis. Филли проверяет кэш перед роутингом — упавший бот не получает запрос.
ГОТОВО
Логирование промахов роутинга
office:routing:misses (LPUSH+LTRIM 100) + log_event route_miss/route_ok в ai_office_shared. Три причины: health_cache_down, http_error, exception. Силли читает через read_logs().
ГОТОВО
Health-cache у Филли
office:health:{agent} в Redis: TTL 60s для up, 120s для down. route_via_http проверяет кэш перед HTTP — упавший бот пропускается сразу без 25с timeout.
ГОТОВО
Cloudflare Worker прокси-резерв
Worker перед Филли. Railway лежит → Worker отвечает "технические работы" вместо тишины. Инцидент 2026-05-20: outage спамил алерты 2 мин/цикл без логики "platform down = заткнись". Уже деплоится на ai-office-watchdog.unperson22.workers.dev.
ГОТОВО
Platform outage detection в watchdog
Перед каждым алертом проверять status.railway.app. Если Major Outage — один алерт и полная тишина до восстановления. Никакого спама.

ЗАДАЧИ — КАНАЛ МАМЫ
Приветствие с инструкцией для мамы
MOM_SYSTEM_BASE + тёплый /start: «Если хотите что-то передать Владу или Луку — просто напишите мне». MOM_USER_ID env (6507691976) задеплоен в Railway mama-bot.
ГОТОВО
mom_queue в Redis
office:mom_queue, LPUSH + TTL 7д. Haiku-классификатор: self | queue | urgent. Urgent — немедленно в OFFICE_CHAT_ID, queue — копит до появления Влада в офисном чате.
ГОТОВО
Естественное упоминание в разговоре
Filly fire-and-forget POST /mention к Эллис при каждом сообщении в группе. Эллис drain()'ит очередь → «💌 Кстати, мама передавала...» в OFFICE_CHAT_ID.
ГОТОВО
Автономное решение мелких вопросов
classify_mom_message() (Haiku): self → Эллис отвечает сама, queue → копит для Влада, urgent → немедленно в офис. Рецепты, вопросы, советы — всё решает сама.
ГОТОВО
Quick Reply кнопки для мамы
InlineKeyboardMarkup: ✅ Да / ❌ Нет / 🕐 Позже / 📞 Позвони. Показываются когда ответ заканчивается вопросом. Кнопка 📞 → mom_queue «Мама просит позвонить».

ЗАДАЧИ — СИЛЛИ 2.0
Claude Code как subagent
Силли вызывает Claude Code CLI для многофайловых рефакторов. CC работает в ветке → PR → ты делаешь /approve → merge. Сейчас Силли правит файлы по одному через GitHub API — синхронные правки контрактов между ботами невозможны без боли.
ГОТОВО
Skills структура (SKILL.md)
Папки ai-office-shared/skills/{name}/SKILL.md с рецептами. Силли видит знакомый сценарий → подгружает скилл → не изобретает заново. Формат тот же что у Anthropic Skills.
ГОТОВО
Базовые скиллы (стартовый набор)
5 скиллов задеплоены в ai-office-shared/skills/: railway-deploy, redis-migration, ptb-reactions, telethon-handlers, github-push. Дальше Силли сам пополняет.
ГОТОВО
Canonical identity registry (tech debt)
ai_office_shared/shared/identity.py: BOTS dict + canonical() + display() + redis_key(). Обратный индекс по всем алиасам (RU/EN, любой регистр). Тэг v0.1.1. Боты переедут при следующей правке shared lib.
ГОТОВО
Shared library в ai-office-shared/shared/
redis_helpers + tasks модули в ai_office_shared v0.1.2. Боты мигрируют попутно при следующей правке (MIGRATION RULE). Написано, протестировано, задеплоено в пакет.
ГОТОВО
Структурные логи в Redis
office:logs:{bot}:{date}, LPUSH+LTRIM 1000, TTL 7д. События: route_*/message_received/response_sent/task_received/api_error/lesson_saved/capability_gap_detected. Пакет ai_office_shared v0.1.0, все 10 ботов.
ГОТОВО
Auto-pull логов при расследовании
Когда Силли ловит баг или capability_gap — сам поднимает последние N логов виновного бота из Redis. Замыкает дебаги → библиотеку → Claude Code в одну цепочку.
ГОТОВО
Автономный ремонт без Railway API
Силли чинит ботов через GitHub + Telegram даже когда Railway API недоступен (platform outage). Паттерн: читает логи из Redis → достаёт скилл → пушит фикс в GitHub → Railway автодеплоит из ветки. Railway API не нужен в цепочке исправления.
